Overview

Base URL
http://localhost:3000/graphql

GraphQL Endpoint

All API requests are made to a single GraphQL endpoint using POST requests with JSON payloads.

Authentication

JWT-based authentication with a complete flow: signup/login to get tokens, use access tokens for protected endpoints like user profile, and refresh tokens when they expire.

Response Format

All responses follow GraphQL conventions with data, errors, and extensions fields.

Authentication

The Cosmic Companion API uses JWT (JSON Web Tokens) for authentication with HTTP-only cookies for refresh tokens. Upon successful signup or login, you'll receive an access token in the response body and a refresh token as an HTTP-only cookie for enhanced security.

Access Token

Short-lived token (12 hours) returned in response body. Include in Authorization header: Bearer <token>

Refresh Token

Long-lived token (7 days) automatically stored as HTTP-only cookie for security. Used automatically by the refresh endpoint.

User Registration

POST /graphql
Register a new user account and receive authentication tokens.

GraphQL Mutation

mutation Register(
    $firstName: String!,
    $lastName: String!,
    $username: String!,
    $email: String!,
    $password: String!
) {
    signup(data: {
        firstName: $firstName,
        lastName: $lastName,
        username: $username,
        email: $email,
        password: $password
    }) {
        token
        refreshToken
        error
    }
}

Parameters

Name Type Required Description
firstName String Required User's first name
lastName String Required User's last name
username String Required Unique username for the account
email String Required Valid email address
password String Required User password (minimum 6 characters recommended)

Example Request

{
    "query": "mutation Register($firstName: String!, $lastName: String!, $username: String!, $email: String!, $password: String!) { signup(data: { firstName: $firstName, lastName: $lastName, username: $username, email: $email, password: $password }) { token refreshToken error } }",
    "variables": {
        "firstName": "Graph",
        "lastName": "QL",
        "username": "graphql",
        "email": "graphql@test.com",
        "password": "GraphQL123"
    }
}

200 OK

Response Fields

Field Type Description

🍪 Refresh Token: The refresh token is automatically set as an HTTP-only cookie for enhanced security. You don't need to handle it manually - the browser will include it in subsequent requests to the refresh endpoint.

Token Refresh

POST /graphql
Refresh an expired access token using either a refresh token parameter or the HTTP-only cookie (recommended).

GraphQL Mutation

# Option 1: Using cookie (recommended)
mutation Refresh {
    refresh {
        token
        error
    }
}

# Option 2: Using token parameter
mutation Refresh($token: String!) {
    refresh(token: $token) {
        token
        error
    }
}

Parameters

Name Type Required Description
token String Optional Refresh token. If not provided, will use the HTTP-only cookie

Example Requests

Option 1: Using Cookie (Recommended)

{
    "query": "mutation { refresh { token error } }"
}

Option 2: Using Token Parameter

{
    "query": "mutation Refresh($token: String!) { refresh(token: $token) { token error } }",
    "variables": {
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MywiZW1haWwiOiJncmFwaHFsQHRlc3QuY29tIiwiaWF0IjoxNzU1NDI2MDE2LCJleHAiOjE3NTYwMzA4MTZ9.hHEwfaZUbASsrmzpMmXYUr4VuMvgktQlMKiraWjF1Hc"
    }
}

Response

200 OK
{
    "data": {
        "refresh": {
            "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MywiZW1haWwiOiJncmFwaHFsQHRlc3QuY29tIiwiaWF0IjoxNzU1NDI4NDkyLCJleHAiOjE3NTU0NzE2OTJ9.JGrjVS9ol4LbPsX7H40deI5teFgFElB6A_T6mQMIi3k",

Response Fields

Field Type Description
token String New JWT access token for API authentication
error String Error message if token refresh failed, null if successful

Usage Notes

🍪 Cookie Method (Recommended): Simply call the refresh mutation without parameters. The HTTP-only cookie will be used automatically.

🔐 Token Parameter Method: Useful for mobile apps or environments where cookies aren't available.

⏰ When to use: Call when your access token expires (typically after 12 hours).

🔄 Best practice: Implement automatic token refresh in your application for seamless user experience.

Get Current User

POST /graphql
Get the current authenticated user's profile information. Requires a valid access token.

GraphQL Query

query {
    me {
        id
        email
        first_name
        last_name
        username
    }
}

Authentication

🔒 Authentication Required: This endpoint requires a valid access token in the Authorization header:

Authorization: Bearer <your-access-token>

Example Request

{
    "query": "query { me { id email first_name last_name username } }"
}

Response

200 OK
{
    "data": {
        "me": {
            "email": "graphql@test.com",
            "first_name": "Graph",
            "last_name": "QL",
            "username": "grapgql",
            "id": "3"
        }
    }
}

Response Fields

Field Type Description
id String Unique user identifier
email String User's email address
first_name String User's first name
last_name String User's last name
username String User's unique username

Error Handling

401 Unauthorized: Missing or invalid access token

403 Forbidden: Token expired - use the refresh endpoint to get a new token
